SnapNote 개인정보 처리방침

SnapNote는 로컬 우선 방식의 가볍고 간결한 메모 앱입니다. 기본적으로 메모 내용과 첨부 파일은 기기에 저장되며, 핵심 기록 기능은 계정 없이 사용할 수 있습니다。

로컬 우선 저장

  • SnapNote는 기본적으로 메모 데이터를 사용자의 기기에 저장하여, 오프라인 상태에서도 기록, 정리, 열람이 가능하도록 설계되었습니다.
  • 현재 버전의 주요 데이터에는 제목, 본문, 태그, 즐겨찾기 상태, 보관 상태 및 관련 로컬 설정이 포함됩니다.

선택형 iCloud 동기화

  • iCloud 동기화는 선택 기능이며, 사용자가 직접 켠 경우에만 SnapNote가 Apple의 iCloud 기능을 사용합니다.
  • 현재 버전에서 주로 동기화되는 항목은 제목, 본문, 태그, 즐겨찾기 상태, 보관 상태 등 텍스트 메모 관련 정보입니다.
  • iCloud를 켜지 않으면 동기화는 시작되지 않으며 앱은 로컬 모드로 계속 동작합니다.

첨부 파일 처리

  • 이미지와 문서 등의 첨부 파일은 기본적으로 기기에 보관되며 iCloud로 자동 업로드되지 않습니다.
  • 이 방식은 로컬 우선 경험을 유지하고 저장 용량, 업로드 동작, 네트워크 사용량을 더 명확하게 관리할 수 있도록 돕습니다.

권한 사용 목적

  • 카메라는 사진을 촬영해 현재 메모의 로컬 첨부 파일로 추가할 때 사용합니다.
  • 사진 권한은 사진 보관함에서 이미지를 선택해 메모에 첨부할 때 사용합니다.
  • 파일 접근은 시스템 파일 선택기를 통해 사용자가 직접 선택한 파일을 가져올 때 사용합니다. SnapNote가 백그라운드에서 파일을 훑어보지 않습니다.
  • 알림은 리마인더, 회고 알림 또는 관련 알림 기능에 사용합니다. 첫 실행 시 한꺼번에 강제로 요청하지 않습니다.
  • Face ID / Touch ID는 앱 잠금을 켜서 기기 내 메모 접근을 보호하는 데 사용합니다. 생체 정보는 시스템이 처리하며 SnapNote가 지문이나 얼굴 데이터를 읽거나 저장하지 않습니다.

필요한 시점에만 권한 요청

  • SnapNote는 앱을 처음 열었을 때 모든 권한을 한 번에 요청하지 않습니다.
  • 촬영, 사진 가져오기, 알림 사용, 앱 잠금 설정처럼 해당 기능을 사용할 때만 관련 권한을 요청합니다.

사용자 제어

  • 카메라, 사진, 알림, 생체 인증 권한은 언제든지 시스템 설정에서 변경할 수 있습니다.
  • 앱 안에서는 iCloud 동기화와 앱 잠금을 끄거나, 필요에 따라 메모와 첨부 파일을 삭제할 수 있습니다.

정보 판매 여부

  • SnapNote는 사용자의 개인정보를 판매하지 않습니다.
  • 현재 버전은 로컬 우선 설계를 따르며, 광고 시스템이나 계정 시스템을 전제로 핵심 메모 기능을 제공하지 않습니다.
